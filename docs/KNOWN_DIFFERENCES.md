# Known differences

- `slip_quadruped/periodic_apex` is now the migrated 22-decision scientific problem. `demo_stride` remains a separate introductory animation and is never used as RoadMap data.
- Residual evaluation wraps cyclic event times deterministically but never performs the source evaluator's hidden `fsolve`. Ground-contact timing projection is an explicit seed operation.
- Public `SimulationResult.Time` is strictly increasing. The source trajectory repeats event timestamps; regression fixtures retain the raw arrays, while the public adapter keeps the last sample at each duplicate time and preserves event states separately.
- The source evaluator returns 23 residual entries for `J_pitch = Inf` and can append optional symmetry equations. The RoadMap contract is finite inertia, no optional symmetry constraints, and exactly 22 residuals.
- The copied FIG files are visual references, not numerical authority. Their Y labels disagree with their row-5 data, and their BG curve predates the current BG MAT branch. The MAT hashes and manifest are authoritative.
- `phi_neutral` is present in the seven-parameter source contract but unused by the active source dynamics. Its schema activity is `inactive`; homotopy rejects it explicitly. Quadruped homotopy/family regressions use nearby active `k_leg` targets and verify that changing the parameter changes the residual before correction.
- Gait classification preserves the source point-classification rules. The compatibility branch helper replaces `downsample` with the equivalent colon index expression, removing its Signal Processing Toolbox-only dependency.
- The renderer recomputes stance leg length so stance feet remain on the ground; swing legs use the resting length. This follows source kinematic equations without migrating its graphics classes.
- `slip_biped/periodic_apex` is the source-equivalent 12-decision/two-offset/15-residual evaluator. Public time removes duplicate event timestamps by keeping the final sample while raw source-regression output retains them; event records preserve pre/post states. Residual entry 12 is the source-reserved zero and is intentionally not removed.
- Biped trajectory fitting preserves the source row-minus-column event-timing implicit expansion (a 5-by-5 norm contribution). The active source `Main.m` path is a penalized objective; an alternate source function exposes the 15 scaled residuals as equality constraints. Both are selectable, and the bounded GUI/example uses the penalized path.
- `slip_quad_load` exposes the exact `44 + 13*(N-1)` source decision rather than the former toy fit. Later-stride additions are nine named events plus four post-contact swing stiffnesses. The public optimizer reduces exact fixed bounds internally but returns the unchanged full vector and full schema.
- The short biped/load optimization examples prove deterministic objective decrease under a bounded evaluation budget; they do not claim a globally optimal biological fit. A budget-limited run can return exit flag 0 even when its objective decreased, and the result records the limit and free indices.
- Canonical IDs are `slip_biped`, `slip_quadruped`, and `slip_quad_load`; deprecated IDs are import aliases only.
- The upstream quadruped repository has no stated license. The biped readme states CC BY-NC 4.0 without a standalone scope-defining license, and the load readme claims BSD-3 while the linked license file is absent at the audited commit. Local authorization does not infer broader redistribution rights; public packaging remains blocked pending the owner decision record.
- Batch graphics, callbacks, recording services, and GUI construction are verified, but the model-specific human-desktop walkthrough and screenshots for all three scientific models, plus MATLAB R2019b execution, remain outstanding. The five retained Round 5 images are automated captures rather than human-desktop evidence.
