# GUI design

`LeggedModelZooApp` owns widgets and rendering handles. Small tab-shell classes (`SimulationTab`, `RoadMapBranchTab`, `SolutionTab`, `SolveTab`, `ContinuationTab`, and `OptimizationTab`) plus reusable problem-badge/inspector components keep construction responsibilities modular. `AppController` owns state transitions and invokes services. `AppState` carries the active scientific catalog, datasets, independent hover and locked selections, working/solved solutions, simulation, seed pair, continuation result/preview, optimization result, oscillator index, recording state, and status.

The application opens on the built-in quadruped RoadMap. Changing models dispatches the same branch explorer to the nine-branch RoadMap, six-branch biped GaitMap, or two built-in load datasets. It can load a manifest selection, all built-ins, a folder, or an individual legacy/native MAT; it supports persisted visibility, remove/reload, Results29/Results14/X_accum or native export, and a details panel for source, status, gait/dataset type, parameter signature, style, point count, and read-only state. It plots schema-named decisions, event times, parameters, or scalar observables with 2-D/3-D, azimuth/elevation, limits, aspect, and model-specific presets.

Hover searches all visible datasets in displayed, range-scaled 2-D/3-D coordinates and writes only `HoverSelection`; a transient marker/data tip shows dataset, index, coordinates, parameters, gait, and residual. Click, keyboard, index, and percentage navigation call `lockBranchPoint`, which synchronizes `Selection` (the Round 4 compatibility alias), `LockedSelection`, `WorkingSolution`, active dataset, problem, and oscillator index. Changing selection or numerical inputs invalidates stale simulation, solve, seed, and continuation state. Source branches are never mutated.

The problem selector displays maturity/validation badges sourced from per-problem catalog descriptors. The inspector separates initial state, event timing, parameters, later-stride additions, observables, residual/objective blocks, diagnostics, and provenance. It displays bounds and active/inactive/derived activity; inactive transport parameters are omitted from homotopy controls with an explanatory tooltip. Editable callbacks update an unevaluated working copy; validate/evaluate repopulates derived data. Restore, simulation, explicit wrap/ground-contact projection, solution/artifact save, writable candidate datasets, solve, continuation, and optimization handoff are controller actions. The GUI contains no evaluator, optimizer, or continuation-engine calls.

`QuadrupedRenderer`, `BipedRenderer`, and `QuadLoadRenderer` consume named kinematics/contact/force data. `AnimationController` maps slider/numeric time to physical frames and reports playback frames back to the UI. Model plot providers draw the appropriate torso/body/leg/load trajectories, footfalls, GRF components, energy/oscillator/tugline and R-squared views in complete or progressive-window mode. `RecorderService` owns atomic GIF/MP4/keyframe/plot/axes exports, cancellation checks, frame restoration, and writer cleanup.

The solve tab supports next/previous and manual RoadMap/GaitMap pairs, generated second seeds, reproducible noise, solve diagnostics, and a pair/predictor overlay. The continuation tab installs prediction/accepted/rejected callbacks for a live source overlay, exposes point count and checkpoint resume, preserves controlled-stop results, and provides active named homotopy/family actions. The optimization tab runs bounded biped/load fits, plots objective history, supports a cancellation request, and retains the optimized solution/result for inspection and save.

Batch construction and callback-level edit/visibility/import/export/controller workflows are automated for all three scientific models. Manual desktop hover ergonomics, OS-native dialogs, human Play/Pause/Stop timing, fit cancellation timing, screenshots, and end-user codec behavior still require a desktop session; `docs/MANUAL_DESKTOP_QA.md` records the blocked checklist.
